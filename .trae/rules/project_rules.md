# 工作区规则（实用版）

本规则聚焦“能跑、好读、可视化清晰、便于学习”，剔除空泛描述，贴合你的使用习惯。

## 目标与范围

- 专注于运筹学演示：线性规划、整数规划、运输问题、网络流、大规模优化、可视化。
- 以教学和演示为主：代码简洁、输出直观、步骤明确。
- 修改不破坏现有功能，接口行为保持一致。

## 输出与清理

- 产出只包含：Python 源码、PNG 图表、`README` 与 `题目集/` 更新。
- 临时测试代码与脚本在验证后立即删除；不保留 `test_*` 文件或临时目录。
- 不留临时数据、缓存或冗余日志；运行后工作区保持干净。
- 图表统一保存为 PNG，`dpi=300`，文件名使用描述性英文或者中文。

## 代码与风格

- 每个 Python 文件必须包含：`#!/usr/bin/env python` 与 `# -*- coding: utf-8 -*-`。
- 中文展示：调用 `font_config.setup_chinese_font()`；数学角标使用 LaTeX（如 `$x_1$`）。
- 统一图表样式：`seaborn-v0_8`；网格 `alpha=0.3`；标题 `fontweight='bold'`。
- 控制台输出使用中文，简洁，突出结论与耗时；可适量使用 emoji 提示。

## 模块结构

- 每个演示模块包含一个主类（驼峰命名，如 `LinearProgrammingDemo`）。
- 方法提供简短中文 docstring：作用、关键约束、目标函数、输入输出。
- 每个模块提供 `main()` 作为独立运行入口。
- 图表与报告保存到项目根目录。

## 工作流程

- 先查找再改动：理解模块结构，定位需要更新的函数或注释。
- 修改后本地运行对应 `main()` 或 `main_demo.py` 验证。
- 验证通过后生成图表/必要文档；随后删除临时测试代码与脚本。
- 更新 `README` 与 `题目集/` 相关段落，保持一致性。

## 语言与语气

- 语言：中文。
- 风格：简明、直接、友好；不堆砌术语，不写空话。
- 语气：面向学习者，给出明确步骤与结论；少叙述，多行动。

## 兼容与性能

- 保持跨平台（Windows/Linux/Mac），优先兼容 Windows 路径与运行方式。
- 演示应在 30 秒内完成；避免过度内存占用。
- 演示场景允许隐藏警告：`warnings.filterwarnings('ignore')`。

## 禁止事项

- 不引入无用依赖，不写空泛段落或重复性说明。
- 不更改现有文件名与接口，除非明确要求。
- 不保留测试残留、临时数据、日志文件或占位注释。
- 不添加版权/许可证头，除非明确要求。

## AI 工作内容清单（定位）

- 统一编码声明与中文 docstring 风格。
- 增改演示模块：建模、求解、可视化、报告生成。
- 生成并保存高质量图表（统一样式与命名）。
- 同步 `README` 与 `题目集/`。
- 验证运行，清理临时测试代码与脚本。
- 只做与当前任务相关的改动，避免扩展范围。

## 交付标准

- 运行成功，无错误，无多余文件或测试残留。
- 图表与输出清晰，中文标签正确，风格统一。
- 规则遵守，改动集中且可读。
